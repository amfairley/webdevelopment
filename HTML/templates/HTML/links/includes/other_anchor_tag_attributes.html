The attributes in the anchor tag provide a better and safer experiences for users of your website. For a comprehensive guide, see the W3Schools webpage for the anchor tag. Here I will cover the common ones that I have employed.

Target

The target attribute specifies where to open the linked document/webpage.

target="_blank"

This opens the link in a new browser tab, which is best for user experience as it keeps the current page open.

target="_self"

This opens the link in the current browser tab, moving away from your webpage. This is the default setting, meaning that if you do not specify the target attribute in your opening anchor tag, it will do this.

Rel

The rel="" attribute specifies the relationship between the current document and the linked document. The most important of these being “noopener”.

rel="noopener"

This prevents malicious behaviour from the destination from impacting your site and is best used for links to social media.

rel="nofollow"

Use this for links to paid links. It helps Google know what parts of your website are yours and what should appear in the searches for your webapp.

rel="help"

For a help document.

rel="next" or rel="prev"

For moving between document selections.

Title

Like a lot of other elements, the anchor tag supports the title attribute that displays some text when the mouse is hovered over it. This is good practice for external links that do not display the website on screen, so that the user can see where they will be redirected.
